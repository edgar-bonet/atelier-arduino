Note: C'est de la scrutation. ## Analogie – scrutation
```c++ void setup() { s_installer_près_du_téléphone(); } void loop() { lire_une_page(); if (le_téléphone_sonne()) { répondre_au_téléphone(); } } ``` Note: Problèmes : * latence * inefficacité On **prête** notre attention au téléphone. ## Analogie – interruption
```c++ void setup() { s_installer_près_du_téléphone(); attachInterrupt(LE_TÉLÉPHONE_SONNE, répondre_au_téléphone); } void loop() { lire_une_page(); } ``` Pré-condition : support matériel dans le processeur. Note: Mécanisme introduit en 1954 d'après Wikipedia Il a existé depuis des processeurs sans interruptions (6507, dans Atari 2600) Le téléphone **attire** notre attention. ## Analogie – chronogrammes Scrutation : ``` début de sonnerie ↓ ┌─────────┬───┬─────────┬───┬─────────┬───┬─────────┬───┬──────────┬─────────┐ │ lire │ é │ lire │ é │ lire │ é │ lire │ é │ répondre │ lire │ └─────────┴───┴─────────┴───┴─────────┴───┴─────────┴───┴──────────┴─────────┘ é = écouter ``` Interruption : ``` début de sonnerie ↓ ┌─────────┬─────────┬─────────┬─────────┬────┬──────────┬─────┬─────────┬─────────┐ │ lire │ lire │ lire │ lire │ li│ répondre │re │ lire │ lire │ └─────────┴─────────┴─────────┴─────────┴────┴──────────┴─────┴─────────┴─────────┘ ``` ## Appel de fonction v.s. interruption Appel de fonction : ```yaml loop: bla bla ma_fonction: bla ┌───▶ bla call ma_fonction ───┘ bla bla ◀─────────┐ bla bla └────────── ret bla ... ``` Interruption : ```yaml loop: bla __vector_42: bla interruption ┌───▶ bla bla ────────────────┘ bla bla ◀─────────┐ bla bla └────────── reti bla ... ``` Note: Registre PC est sauvegardé/restauré de la même manière. L'appel de fonction arrive au moment prévu par le programmeur. L'interruption peut arriver avant n'importe quelle instruction. ## Logique de déclenchement d'une interruption ![Schéma logique du contrôle de l'interruption](img/int-logique.png) Note: L'ordre d'allumage des bits n'est pas important. Les interruptions ne sont pas interruptibles → différence `ret`/`reti` Notion d'interruption en attente. ## Mise en œuvre des interruptions – via Arduino core Interruptions externes : le core fournit la fonction [attachInterrupt()][]. ```c++ const uint8_t broche_bouton = 2; const uint8_t numero_interruption = digitalPinToInterrupt(broche_bouton); // Gestionnaire d'interruption. void mettre_a_jour_led() { digitalWrite(LED_BUILTIN, digitalRead(broche_bouton)); } void setup() { pinMode(broche_bouton, INPUT); pinMode(LED_BUILTIN, OUTPUT); attachInterrupt(numero_interruption, mettre_a_jour_led, CHANGE); } void loop() {} ``` **Exercice**: Utiliser des interruptions séparées pour allumer et éteindre la LED (modes `RISING` et `FALLING`). [attachInterrupt()]: https://docs.arduino.cc/language-reference/en/functions/external-interrupts/attachInterrupt/ Note: `01_led-bouton.ino` ## Mise en œuvre des interruptions – via une bibliothèque Gestionnaire de bibliothèques : téléchager [TimerOne][]. ```c++ #include // Gestionnaire d'interruption. void commuter_LED() { static uint8_t etat_LED = LOW; if (etat_LED == LOW) { etat_LED = HIGH; } else { etat_LED = LOW; } digitalWrite(LED_BUILTIN, etat_LED); } void setup() { pinMode(LED_BUILTIN, OUTPUT); Timer1.initialize(400000); // période = 400 000 µs = 0,4 s Timer1.attachInterrupt(commuter_LED); } void loop() {} ``` **Exercice**: Installer TimerOne et tester ce programme. [TimerOne]: https://www.pjrc.com/teensy/td_libs_TimerOne.html Note: `02_timer-one.ino` ## Mise en œuvre des interruptions – façon « bare metal » ```c++ const char message[] = "Lorem ipsum dolor sit amet, consectetur " "adipiscing elit. Pellentesque euismod, velit ut rutrum faucibus, " "ipsum lorem sodales magna, vitae feugiat est ac risus."; // Gestionnaire de l'interruption « transmit complete ». ISR(USART_TX_vect) { digitalWrite(LED_BUILTIN, HIGH); } void setup() { Serial.begin(9600); pinMode(LED_BUILTIN, OUTPUT); UCSR0B |= bit(TXCIE0); // activer interruption USART_TX } void loop() { Serial.println(message); delay(500); digitalWrite(LED_BUILTIN, LOW); } ``` Il faut lire [la fiche technique du microcontrôleur][datasheet] (seulement 653 pages). [datasheet]: https://ww1.microchip.com/downloads/aemDocuments/documents/MCU08/ProductDocuments/DataSheets/ATmega48A-PA-88A-PA-168A-PA-328-P-DS-DS40002061B.pdf Note: `03_flash-apres-tx.ino` Deux étapes: * définir l'ISR avec le bon nom * activer l'interruption via les registres de contrôle Utile pour contrôler ligne DE d'un transceiver RS-485. ## Implémentation de attachInterrupt() D'après [ArduinoCore-avr][]: ```c++ static void nothing() {}; volatile voidFuncPtr intFunc[2] = { nothing, nothing }; ISR(INT0_vect) { intFunc[0](); } ISR(INT1_vect) { intFunc[1](); } void attachInterrupt(uint8_t interruptNum, void (*userFunc)(void), int mode) { intFunc[interruptNum] = userFunc; switch (interruptNum) { case 0: EICRA = (EICRA & ~(bit(ISC00) | bit(ISC01))) | (mode << ISC00); EIMSK |= bit(INT0); break; case 1: EICRA = (EICRA & ~(bit(ISC10) | bit(ISC11))) | (mode << ISC10); EIMSK |= bit(INT1); break; } } ``` [ArduinoCore-avr]: https://github.com/arduino/ArduinoCore-avr/blob/1.8.6/cores/arduino/WInterrupts.c ## Interruptions du Timer 0 On peut avoir « gratuitement » des interruptions toutes les 1 024 µs : TIMER0\_COMPA (PWM 6) et TIMER0\_COMPB (PWM 5). ```c++ bool fini = false; // est-ce que l'attente est finie ? // Gestionnaire d'interruption appelé toutes les 1,024 ms. ISR(TIMER0_COMPA_vect) { fini = true; } void setup() { Serial.begin(9600); Serial.println("En attente..."); TIMSK0 |= bit(OCIE0A); // activer l'interruption TIMER0_COMPA while (!fini) { /* attendre... */ } Serial.println(" fini!"); } void loop() {} ``` **Exercice**: tester ce programme. Note: `04_timer-0.ino` Optimisation 1 : ```c register bool fini_copie = fini; while (!fini_copie) { ... } ``` Optimisation 2 (inutile d'en parler) : `if (!fini) while (true) {}` Solution : `volatile bool fini = false;` ## Durée d'exécution d'une ISR ```c++ const uint32_t periode_de_commutation = 400; // Gestionnaire d'interruption sur conversion ADC terminée. ISR(ADC_vect) { delayMicroseconds(0); } void setup() { pinMode(LED_BUILTIN, OUTPUT); ADCSRA |= bit(ADIE); // activer l'interruption ADC } void loop() { static uint32_t temps_dernier_changement = 0; static uint8_t etat_LED = LOW; if (millis() - temps_dernier_changement >= periode_de_commutation) { temps_dernier_changement += periode_de_commutation; etat_LED = etat_LED==LOW ? HIGH : LOW; digitalWrite(LED_BUILTIN, etat_LED); } analogRead(A0); // provoque l'exécution de ISR(ADC_vect) delayMicroseconds(0); } ``` **Exercice**: varier tour à tour chacun des délais. À partir de quand le rythme de la LED est visiblement perturbé ? Essayer ≥ 1 000 µs. Note: `05_duree-isr.ino` ## Cohérence des données ```c++ volatile uint32_t counter; ISR(TIMER0_COMPA_vect) { if (counter == 0xffffffff) counter = 0x00000000; else counter += 0x01010101; } void setup() { TIMSK0 |= bit(OCIE0A); Serial.begin(9600); Serial.println("Ready"); } void loop() { union { uint32_t value; uint8_t bytes[4]; } data; data.value = counter; for (int i = 1; i < 4; i++) { if (data.bytes[i] != data.bytes[0]) { Serial.println(data.value, HEX); break; } } delayMicroseconds(500); } ``` **Exercice**: Tester et corriger ce programme. Notion d'accès « atomique » = « insécable » Notion de « section critique ». ## Réentrance Les fonctions manipulant des ressources globales sont non réentrantes : **Fonctions manipulant** : * des variables globales * des variables locales `static` **Fonctions de bibliothèque** : * opérateurs `new`, `delete` * `rand()`, `srand()`, `random()`, `randomSeed()` * `strtok()` * fonctions modifiant `errno` * tout ce qui accède au matériel (`Serial.*()`...) * ... ## Résumé * Les gestionnaires d'interruption doivent: * être courts * appeler uniquement des fonctions réentrantes * les variables partagées avec le reste du programme doivent: * être déclarées `volatile` * être accédées de façon atomique (section critique si nécessaire) ⇒ Préférer la scrutation quand c'est possible.
